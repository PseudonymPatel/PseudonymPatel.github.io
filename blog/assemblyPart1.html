Learning Assembly in 2020 but it’s the 80s - Part 1

Hi there! This is going to be a series of blog posts where you can follow my journey of learning assembly, but with a twist: I have to do it without the internet.

Why Assembly?

Before I get into the process, I want to address something first: Who learns assembly in 2020? It’s such a niche topic and few programmers ever even interact with it directly. Maybe some legacy programmers use it in a job; definitely the security analysts know it. But no “mainstream” programming job ever uses it. Maybe you might see it in the debugger as you debug the bug you’ve been working on for the last couple weeks. (Hint: take a break; it’ll come to you in a dream.)

So that begs the question, why the hell do I want to learn assembly> of all languages? Well, it’s been on my mind for a while, especially as I’ve dabbled in some CTFs lately. I love learning the underlying stuff that makes things tick and the CPU fan go brrr. Assembly seemed like that perfect mix of something novel but still useful.

With a twist

Of course, nothing I do can ever be straightforward. That’s why I decided to give myself a bit of a challenge embarking on this one. I decided I would learn it like it’s the 80s. No internet. No help. Just the processor manual. Yes, that means no StackOverflow. Why, you may ask? I dunno. It seems like a good idea, and I’m taking my time anyways. It’s the year before college, and I thought that I should sneak in something new throughout the year (after college apps, of course). I’ve always moaned how programming has gotten so easy nowadays and people are learning to program inefficiently and bassackwards, so I thought I’d experience the old ways.

Planning it out

My computer runs a Ryzen 5 2600, so naturally I’d use AMD’s processor specification/manual thing. It doesn’t really matter where I learn assembly from, as standardization nowadays makes things essentially the same for my purposes. After 5 minutes of Googling, I found the “AMD64 Architecture Programmer’s Manual”. I just downloaded all 5 of them, they seem to build on one another.

https://developer.amd.com/resources/developer-guides-manuals/ All the AMD manuals and resources I will use can be found here.

Armed with the manual, I also had to figure out how to run the assembly code I would write. To make things easier, I would use neovim (basically vim for those unfamiliar) as an editor and Ubuntu as an operating system. As a compiler, I’d use one of those _ASM assemblers. NASM and FASM seemed like the two big ones people talked about, and I decided to go with FASM as the good folks on StackOverflow put in some good words about it and it seems like it’s able to compile for a wide range of operating systems (not that it matters, from my understanding the instruction sets are essentially the same, and I won’t be using any extensions or virtualization stuff).

After an hour or two of research into assembly and machine code, I will probably use the internet for learning how to use FASM to compile programs. Before you shout SCAM! SCAM!, I will still learn the actual assembly part using solely the architecture manuals. Also, from my understanding before embarking on this, the manual seems to be talking about the machine code part of it, but they also give mnemonics (the human-readable names for instructions). I am assuming that FASM uses the same mnemonics. If they don’t, anyone reading this will be cringing in their seat! (It will also be discussed in depth in the next part.)

As I’m doing research, I’m also finding things like syscall, which seem to be calling the system operating system and does things like output and exiting. For these things, I don’t know if they are part of the AMD manual or the operating system. If it is based on the operating system, I will try to stay true to the idea of this challenge and only use the manual and not forums or StackOverflow.

A revised plan

With the above considerations in mind, here is a summary of the challenge:

  • I will be using the x86_64 AMD Architecture with the Ubuntu operating system.
  • I will use FASM as an assembler and neovim as a text editor.
  • No internet will be allowed*, except to get the necessary manuals.
    • *I may have to use the FASM website to learn how to use FASM, but that is all.
  • The overarching idea is to stay true to old-fashioned 80s programming on modern hardware, and if there is any conflicts in the rules I set up for myself, the more “authentic” choice will be taken.

Quick Update

A "programmer's manual" is provided with the fasm assembler. I will use that as a reference and to help me learn things like mnemonics. I will also use other files that come with the installation of fasm, as this would fit with the style of the challenge.
